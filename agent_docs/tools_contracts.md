# Tool / Function Contracts (Prototype)

In this prototype, “tools” are local functions called by the triage engine.
They must be:
- deterministic
- side-effect free unless explicitly a logging/persist tool
- well-typed (Python types) and covered by simple unit tests where relevant

## Read-only “data tools”
- get_case(case_id) -> MockCase
- get_customer_profile(customer_id) -> dict
- get_tx_history_aggregates(customer_id) -> dict
- get_device_context(device_id) -> dict
- check_sanctions_pep(customer) -> { hit: bool, match_type: str }
- link_entities(case) -> { clusters: [...], ring_suspicions: [...] }

## Write “tools”
- write_triage_results(result_obj) -> None (writes out/triage_results.json)
- append_audit_log(audit_row) -> None (writes out/audit_log.jsonl)
- append_override_log(override_row) -> None (writes out/overrides.jsonl)

## LLM narrative tool
- generate_narrative(evidence_pack, decision_fields) -> string

Constraints:
- Only narrative text is generated by LLM.
- The prompt must instruct “use only provided fields, do not infer facts.”
- Free-text inputs are untrusted and must be clearly labeled as such.
